水平格式化

水平格式化包括 margin-left border-left padding-left width padding-right border-right margin-right 7个属性

其中只有三个属性可以设置为auto，那就是width、margin-left margin-right，其他属性必须设置为特定的值

使用auto

由于只有margin-left、width、margin-right可以设置为auto

那么当其中两个值设置为具体值，而另外一个值设置为auto事，设置为auto的那个值会自动计算

父元素width设置为400px

子元素margin-right设置为100px，width设置为100px，margin-left设置为auto

此时margin-left会自动计算(假设没有padding和border)，400 - width - margin-right

400 - 100 - 100 = 200，此时margin-left: auto的值计算为20ppx

margin-left width margin-right都设置为具体值

如果这三个值都设置为具体值，那么会强制的将maring-right设置为auto

强制的使元素的总宽度等于其包含块的宽度

其实这个也很好理解，正常文档流是从左到右的

如果设置了margin-left,就已经可以确定元素的位置，而不需要margin-right

margin-left和margin-right设置具体值，width: auto自动计算

margin-left和margin-right设置为auto，左右外边距等分， width设置为具体值

三个值都设置为auto

答案就是两个外边距都会为0，而width相当于auto

也就相当于默认的值，margin-left和margin-right默认是0，width默认是auto

负外边距

不设置border、padding、的情况下

margin-left + width + margin-right的值等于父元素的宽度

这三个值又能设置为auto自动计算，margin有能设置为负值

那我们给其中一个margin负值，那是不是width的auto值会越来越大呢,甚至超出父元素设置的width

可以看到inner元素的width等于400，并且它还带了100的左那边距

虽然元素内容超出了父元素的width

但是 100 + -100 + 400 = 400,这个是没有错的

设置marign-left为负值，元素左边增大

设置margin-right为负值，元素右边增大

width设置为大于包含块宽度，margin-right设置为auto

并且子元素的margin-right auto计算为-100

width设置具体值，margin-left设置为负数，元素向左移动，但是不改变元素的宽度

在正常流中，从左到右

margin-left为正值，元素就会往右偏移

margin-left为负值，元素就能往左偏移

左边元素简称a右边简称b

左边元素设置margin-left正数，ab元素都会往右移动

左边元素设置margin-left负数,ab元素往右移动

左边元素设置margin-right正数，b元素右移动，a不变

左边元素设置margin-right负数，a不变，b右移

正常流中，从左到右，从上到下

margin-left会影响自己本身元素的位置，从影响后面元素的位置

margin-right 因为是从左到右，margin-right可以理解为不会改变自身元素的位置

但是对于下一个元素来说，这个margin-right相当于设置了下一个元素的margin-left

a的margin-right，相当于b的margin-left

从而a的margin-right不会影响自身

margin-top和margin-bottom也是如此
